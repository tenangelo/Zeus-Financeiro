FLUXO DE DADOS (Semana 2)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

╔════════════════════════════════════════════════════════════════════════════╗
║                          PARSER NFCe WORKFLOW                             ║
╚════════════════════════════════════════════════════════════════════════════╝

Usuário (PDV)
    │
    └─→ Emite NFC-e (XML) → arquivo.xml
        │
        └─→ Upload via: POST /import/upload-nfce
            │
            ├─→ Controller recebe arquivo
            │   │
            │   └─→ NFCeParser.parse(xmlBuffer)
            │       │
            │       ├─1. Converter XML → Objeto JS (xml2js)
            │       ├─2. Extrair dados (chaveNFe, data, itens, valor)
            │       ├─3. Validar (status = "autorizado"?)
            │       └─4. Retornar NFCeData ou null
            │
            ├─→ NFCeParser.toTransaction(nfce, tenantId, userId)
            │   │
            │   └─→ { type: "income", amount: 125.50, ... }
            │
            └─→ INSERT INTO transactions (...)
                │
                └─→ RLS valida: tenant_id matches ✅
                    │
                    └─→ Dados no banco! ✅


╔════════════════════════════════════════════════════════════════════════════╗
║                          PARSER CSV WORKFLOW                              ║
╚════════════════════════════════════════════════════════════════════════════╝

Usuário (PDV)
    │
    └─→ Exporta CSV → vendas.csv (de Saipos, GrandChef, etc)
        │
        └─→ Upload via: POST /import/upload-csv
            │
            ├─→ Controller recebe arquivo
            │   │
            │   ├─→ CSVParser.detectTemplate(csvBuffer)
            │   │   │
            │   │   ├─1. Ler headers
            │   │   ├─2. Contar quantas colunas conhecidas
            │   │   └─3. Retornar melhor template (saipos/grandchef/generic)
            │   │
            │   └─→ CSVParser.parse(csvBuffer, template)
            │       │
            │       ├─1. Parsear CSV (papaparse)
            │       ├─2. Mapear colunas (Data → transaction_date, etc)
            │       ├─3. Normalizar datas (DD/MM/YYYY → YYYY-MM-DD)
            │       ├─4. Normalizar valores (vírgula → ponto)
            │       └─5. Retornar array de rows
            │
            ├─→ Para cada row:
            │   │
            │   └─→ CSVParser.toTransaction(row, tenantId, userId)
            │       │
            │       └─→ { type: "income"/"expense", amount: 45.50, ... }
            │
            └─→ INSERT INTO transactions (...) [batch]
                │
                └─→ RLS valida: tenant_id matches ✅
                    │
                    └─→ Dados no banco! ✅
